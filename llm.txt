MazadAuto — Enchères Automobile en Tunisie
=============================================

URL: https://mazadauto.tn
Stack: Next.js 16.2.6 (Turbopack), TypeScript strict, Tailwind CSS v4, Framer Motion
Base de données: Supabase (PostgreSQL + Auth + Realtime + Storage)
Notifications: Resend (emails), Twilio (SMS), Supabase Realtime (in-app)
CI/CD: Vercel

Architecture
------------
- App Router (Next.js 16). 35 pages statiques + API routes dynamiques.
- `app/(marketing)/` — Landing page (Navbar, Hero, Live Auctions, How It Works, Trust, CTA, Footer)
- `app/(auth)/` — Connexion, inscription (acheteur/vendeur), wizard 5 étapes, paiement empreinte 600 DT
- `app/encheres/` — Listing avec filtres, tri, pagination (client-side, URL search params)
- `app/encheres/[slug]/` — Fiche véhicule (SSG, generateStaticParams), OG image dynamique, JSON-LD
- `app/dashboard/` — Dashboards acheteur/vendeur/admin (données statiques/mockées en MVP, à connecter à Supabase en production)
- `app/api/og/` — Open Graph image dynamique 1200×630 PNG
- `app/sitemap.ts`, `app/robots.ts` — SEO

Données
-------
- Mock fallback: `lib/data/mock-vehicles.ts` (10 véhicules complets avec inspections, photos, enchères, vendeurs).
- Quand Supabase est configuré et seedé (001_initial_schema.sql + 002_seed_data.sql), les pages basculent automatiquement sur les vraies données.

Authentification & KYC
----------------------
- Inscription: email/password via Supabase Auth.
- KYC: upload CIN (recto/verso), selfie, RIB, patente/statuts selon type de compte.
- Validation manuelle par admin (24-48h).
- Empreinte bancaire 600 DT requise pour enchérir.

Mode Test (E2E manuel)
-----------------------
- Bouton "Mode Test" dans la navbar du landing page.
- Active un cookie `__test_mode=true` + localStorage.
- Bypass complet de l'authentification dans le middleware pour `/dashboard/*`.
- Permet de tester les enchères, les dashboards, les profils sans login.
- Bannière jaune persistante sur toutes les pages quand actif.
- Peut être supprimé sur demande.

Filtres enchères (URL search params)
------------------------------------
- Marque (radio, liste dynamique depuis les données)
- Carburant: essence | diesel | hybride | electrique | gpl
- Boîte: manuelle | automatique
- Prix min/max (en milliers de DT, filtre sur current_price)
- Kilométrage max
- Année min
- Tri: plus récentes / se terminent bientôt / prix croissant / décroissant

SEO
---
- Metadata root avec OG/Twitter/JSON-LD.
- `generateMetadata` dynamique sur chaque fiche véhicule.
- `/api/og` génère une image 1200×630 avec titre + prix.
- Sitemap 14 URLs, robots.txt.
- Structured data: Car + Organization schema.

Edge Functions (Supabase)
-------------------------
- `check-ending-auctions` — notifie les enquêteurs/watchers quand <1h restante.
- `finalize-ended-auctions` — détermine gagnant/perdant, crée notifications.
- Déclenchées par pg_cron toutes les 5 minutes.

Production Hardening
--------------------
- Zod validation sur auth, vehicle, auction, bid.
- Rate limiting placeholder (Upstash Redis, commenté — activer quand env vars configurées).
- Security headers dans middleware (X-Frame-Options, HSTS, etc.).
- Validation uploads: MIME whitelist, 5MB max, magic bytes.
- Sentry configs commentées (client/server/edge).
- Error boundaries: `error.tsx`, `not-found.tsx`.
- Playwright E2E: 4 spec files + helpers (commentés — décommenter après `npm install -D @playwright/test`).

Conventions
-----------
- Pas de UI library (shadcn/MUI) — composants custom Tailwind uniquement.
- Couleurs: fond blanc `#ffffff`, accent vert `#16a34a`, typographies Bricolage Grotesque + Inter.
- Mobile-first, fully responsive.
- Tout le texte en français, prix en DT (milliers).
- `data-testid` sur les éléments critiques pour E2E.

Suppressions demandées (en attente)
-----------------------------------
- Mode Test: à supprimer quand l'utilisateur le demande via un prochain prompt.

Points d'attention connus
-------------------------
- Les dashboards (acheteur/vendeur/admin) contiennent actuellement des données statiques/mockées. Ce sont des placeholders UI pour le MVP. À connecter à Supabase en production.
- React Email est remplacé par des templates HTML string (`lib/email/templates.ts`) car l'install a été interrompue dans cet environnement. Réinstaller `@react-email/components` et réactiver les imports dans `lib/resend/client.ts` pour revenir à React Email.
- `date-fns` est remplacé par un helper `timeAgo()` custom (même raison).
- Upstash rate limiting et Sentry sont des placeholders commentés.
